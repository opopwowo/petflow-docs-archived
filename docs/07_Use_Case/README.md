# Use Case 使用案例

> 描述參與者與系統互動的完整案例。

## 模組介紹

本模組以使用案例規格記錄主要流程、替代流程與例外情境，明確參與者（Actor）、前置條件、後置條件，補足 User Story 的互動細節。

## 目的

- 定義系統邊界與參與者（Actor）
- 撰寫 Use Case 規格（主流程 / 替代 / 例外）
- 繪製 Use Case Diagram
- 對應 Use Case 至 API 與流程圖

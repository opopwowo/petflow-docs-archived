# 前置／後置條件清單

> 彙整全模組 Use Case 的前置條件與後置條件對照表，將登入、租戶、權限、方案限制與資料狀態等條件標準化編碼，供 API 設計與測試案例直接引用。

## 1. 使用說明

- 本文件與 [02_各模組Use_Case規格書](02_各模組Use_Case規格書.md) 一一對應；規格書為敘述式，本文件為結構化對照表。
- 前置條件以**標準條件代碼**（第 2 節）表示，各 UC 僅另列「特定前置條件」；違反前置條件時的系統行為見 [04_例外與錯誤情境彙整](04_例外與錯誤情境彙整.md)。
- 後置條件僅列成功路徑保證成立的狀態；寫入類 UC 一律隱含「已寫入 Audit Log」，表中以 ✔（AUD）簡記。

## 2. 標準前置條件代碼

### 2.1 登入與租戶（PRE-AUTH／PRE-TNT）

| 代碼 | 條件 | 違反時行為 |
| --- | --- | --- |
| PRE-AUTH-01 | 已登入且 Token 有效 | 401（EXC-01） |
| PRE-AUTH-02 | Token 內含有效租戶上下文 | 401／403（EXC-01／02） |
| PRE-TNT-01 | 租戶為啟用狀態（非停用／凍結） | 403（EXC-02） |
| PRE-TNT-02 | 目標資源屬於本租戶 | 404 隱藏存在性（EXC-03） |
| PRE-TNT-03 | 平台層身分（SUPER_ADMIN） | 403（EXC-02） |

### 2.2 權限（PRE-RBC）

| 代碼 | 條件 | 違反時行為 |
| --- | --- | --- |
| PRE-RBC-01 | 具備該 UC 宣告之權限（`資源:動作`，Deny by default） | 403（EXC-02） |
| PRE-RBC-02 | 操作對象角色層級不高於操作者自身 | 403（EXC-02） |
| PRE-RBC-03 | VET 僅限被授權寵物之健康資料（FR-HLT-009） | 403（EXC-02） |
| PRE-RBC-04 | 使用者門市範圍涵蓋目標資料（P1，FR-STO-003） | 404 隱藏存在性（EXC-03） |

### 2.3 方案限制（PRE-SUB）

| 代碼 | 條件 | 違反時行為 |
| --- | --- | --- |
| PRE-SUB-01 | 寵物數未達方案額度 | 422 額度超限（EXC-07） |
| PRE-SUB-02 | 使用者數未達方案額度 | 422 額度超限（EXC-07） |
| PRE-SUB-03 | 照片容量與單檔大小未超過方案限制 | 422 額度超限（EXC-07） |
| PRE-SUB-04 | 門市數未達方案額度（P1） | 422 額度超限（EXC-07） |
| PRE-SUB-05 | 租戶不處於到期寬限唯讀模式 | 422 提示續約（EXC-07） |
| PRE-SUB-06 | 方案包含目標功能（如自訂角色屬 Enterprise） | 422 提示升級（EXC-07） |

### 2.4 資料狀態（PRE-DAT）

| 代碼 | 條件 | 違反時行為 |
| --- | --- | --- |
| PRE-DAT-01 | 目標資源存在且未被軟刪除 | 404（EXC-04） |
| PRE-DAT-02 | 目標資源處於已軟刪除狀態（還原類 UC 專用） | 409（EXC-05） |
| PRE-DAT-03 | 目標資源處於狀態機允許的來源狀態 | 409 非法狀態轉移（EXC-05） |
| PRE-DAT-04 | 唯一性欄位不與同租戶既有資料衝突（晶片號碼等） | 409（EXC-05） |
| PRE-DAT-05 | 版本標記一致（無併發修改） | 409 併發衝突（EXC-08） |
| PRE-DAT-06 | 關聯資料齊備（如寵物已有主要飼主、範本存在） | 422（EXC-06） |

## 1. 使用說明

- **編號**：`US-<模組代碼>-NNN`，同模組內不重複、不回收。
- **角色**：一律使用 [05 使用者角色](../05_使用者角色/01_Persona卡片.md) 之 Persona——阿豪／志明（OWNER）、雅婷（ADMIN，兼 MANAGER 代管情境）、小美（STAFF）、Dr. Chen（VET）、宥廷（SUPER_ADMIN）、VIEWER（唯讀）。

## 2. 寵物管理（PET）

### US-PET-001 建立寵物檔案

> 身為小美（店員／STAFF），我想要在寵物到店時快速建立寵物檔案（Pet），以便所有後續照護與買賣作業都有可追溯的主檔。

- **優先級**：P0｜**對應 FR**：FR-PET-001、FR-PET-002

**驗收條件**

1. Given 小美已登入且具 `pet:create` 權限，When 她輸入名稱與物種並送出，Then 系統建立寵物檔案、回傳 201，且列表可見該寵物。
2. Given 小美未填寫必填欄位（名稱或物種），When 送出表單，Then 系統回傳 422 並逐欄提示缺漏，不建立任何資料。
3. Given 同租戶內已存在相同晶片號碼之寵物，When 小美以該晶片號碼建立寵物，Then 系統回傳 409 並顯示既有寵物連結，不建立重複檔案。

### US-PET-002 搜尋與過濾寵物列表

> 身為小美（店員／STAFF），我想要以關鍵字、狀態與排序條件查詢寵物列表，以便在客人詢問時數秒內找到目標寵物。

- **優先級**：P0｜**對應 FR**：FR-PET-003

**驗收條件**

1. Given 租戶內存在多筆寵物，When 小美以名稱、晶片號碼或品種關鍵字搜尋，Then 僅回傳本租戶符合條件的寵物並支援游標分頁。
2. Given 小美選擇狀態過濾（如「在店」），When 套用過濾，Then 列表僅顯示該狀態且未被軟刪除的寵物。

### US-PET-003 檢視寵物詳情

> 身為雅婷（租戶管理員／ADMIN），我想要在單一詳情頁看到寵物的健康摘要、封面照、飼主關聯與登記案件狀態，以便不用在多個頁面間切換即可掌握全貌。

- **優先級**：P0｜**對應 FR**：FR-PET-004

**驗收條件**

1. Given 寵物已有疫苗紀錄、封面照、飼主與登記案件，When 雅婷開啟詳情頁，Then 頁面整合顯示健康摘要、封面照、飼主清單與各登記案件目前狀態。
2. Given 雅婷嘗試以其他租戶之寵物 ID 開啟詳情頁，When 系統驗證租戶上下文，Then 回傳 404，不洩漏該資源存在性。

### US-PET-004 編輯寵物資料

> 身為小美（店員／STAFF），我想要修改寵物的基本資料，以便資料異動（如體重、毛色、備註）能即時反映在系統中。

- **優先級**：P0｜**對應 FR**：FR-PET-005

**驗收條件**

1. Given 小美具 `pet:update` 權限，When 她修改寵物欄位並儲存，Then 變更立即生效，且 Audit Log 記錄操作者、時間與 before/after 內容。
2. Given 小美將晶片號碼改為租戶內既有值，When 儲存，Then 系統回傳 409 且不套用任何變更。

### US-PET-005 管理寵物狀態

> 身為小美（店員／STAFF），我想要將寵物狀態在「在店／已售／寄養／歿」之間切換並保留歷程，以便店內即時掌握每隻寵物的去向。

- **優先級**：P0｜**對應 FR**：FR-PET-006

**驗收條件**

1. Given 寵物目前狀態為「在店」，When 小美將狀態改為「已售」，Then 狀態更新成功且狀態歷程新增一筆含時間與操作者的紀錄。
2. Given 使用者檢視寵物詳情頁，When 展開狀態歷程，Then 可依時間倒序看到全部狀態變更紀錄。

### US-PET-006 軟刪除與還原寵物

> 身為雅婷（租戶管理員／ADMIN），我想要將建錯的寵物檔案移入回收區並可還原，以便誤刪時不會永久遺失資料。

- **優先級**：P0｜**對應 FR**：FR-PET-007

**驗收條件**

1. Given 雅婷具 `pet:delete` 權限，When 她刪除寵物，Then 寵物被標記 `deleted_at`、從所有預設查詢消失，並可於回收區檢視。
2. Given 寵物位於回收區，When 雅婷執行還原，Then 寵物恢復於列表與詳情頁，且刪除與還原均寫入 Audit Log。
3. Given 小美（STAFF，無刪除權），When 她嘗試刪除寵物，Then 系統回傳 403 且資料不受影響。

### US-PET-007 維護品種主檔

> 身為雅婷（租戶管理員／ADMIN），我想要使用內建犬貓品種清單並可新增自訂品種，以便建檔時品種名稱一致、可統計。

- **優先級**：P0｜**對應 FR**：FR-PET-009

**驗收條件**

1. Given 系統內建常見犬貓品種，When 小美建立寵物並輸入品種，Then 可從清單挑選且支援關鍵字快速篩選。
2. Given 清單中無所需品種，When 雅婷新增自訂品種，Then 該品種僅於本租戶可見並可立即用於建檔。

### US-PET-008 記錄體重趨勢

> 身為 Dr. Chen（特約獸醫／VET），我想要為寵物記錄多筆體重並檢視趨勢圖，以便追蹤發育與健康狀況。

- **優先級**：P1｜**對應 FR**：FR-PET-008

**驗收條件**

1. Given 寵物已有多筆體重紀錄，When Dr. Chen 開啟體重頁籤，Then 以時間序趨勢圖與列表呈現全部紀錄。
2. Given Dr. Chen 新增一筆體重，When 儲存成功，Then 趨勢圖即時更新且紀錄寫入 Audit Log。

### US-PET-009 批次匯入寵物資料

> 身為阿豪（單店老闆／OWNER），我想要以 CSV 範本批次匯入既有寵物資料，以便從 Excel 轉移到系統時不用逐筆重打。

- **優先級**：P1｜**對應 FR**：FR-PET-010

**驗收條件**

1. Given 阿豪下載官方 CSV 範本並填妥資料，When 上傳匯入，Then 系統逐列檢核並回報成功筆數與逐列錯誤報告（含列號與原因）。
2. Given 匯入檔中部分列格式錯誤，When 執行匯入，Then 正確列成功建立、錯誤列不寫入，且不產生半套資料。

## 3. 飼主管理（OWN）

### US-OWN-001 建立飼主檔案

> 身為小美（店員／STAFF），我想要建立飼主（Owner）檔案並在輸入電話時偵測重複，以便避免同一位客人被重複建檔。

- **優先級**：P0｜**對應 FR**：FR-OWN-001、FR-OWN-002

**驗收條件**

1. Given 小美具 `owner:create` 權限，When 她輸入姓名與電話並送出，Then 系統建立飼主檔案並回傳 201。
2. Given 租戶內已有相同電話之飼主，When 小美輸入該電話，Then 系統即時提示既有飼主並可一鍵選用，不強制建立新檔。
3. Given 必填欄位（姓名或電話）缺漏，When 送出，Then 回傳 422 並逐欄提示。

### US-OWN-002 搜尋飼主

> 身為小美（店員／STAFF），我想要以姓名或電話搜尋飼主並依標籤過濾，以便客人來電時立即調出資料。

- **優先級**：P0｜**對應 FR**：FR-OWN-003

**驗收條件**

1. Given 租戶內存在多筆飼主，When 小美以姓名或電話片段搜尋，Then 僅回傳本租戶符合條件且未被軟刪除的飼主，支援分頁。
2. Given 飼主已被標記標籤（如 VIP），When 以標籤過濾，Then 僅顯示含該標籤之飼主。

### US-OWN-003 檢視飼主詳情與名下寵物

> 身為雅婷（租戶管理員／ADMIN），我想要在飼主詳情頁看到名下所有寵物與互動紀錄，以便快速掌握客戶全貌。

- **優先級**：P0｜**對應 FR**：FR-OWN-004

**驗收條件**

1. Given 飼主名下有多隻寵物，When 雅婷開啟飼主詳情頁，Then 顯示全部關聯寵物（含主要飼主標記）與互動紀錄時間軸。
2. Given 名下寵物已被軟刪除，When 檢視詳情頁，Then 該寵物預設不顯示於名下清單。

### US-OWN-004 建立寵物—飼主關聯

> 身為小美（店員／STAFF），我想要將寵物與多位飼主建立關聯並指定主要飼主，以便正確反映共同飼養的實際情形。

- **優先級**：P0｜**對應 FR**：FR-OWN-005

**驗收條件**

1. Given 寵物與飼主均存在於本租戶，When 小美建立關聯並指定其中一位為主要飼主，Then 寵物詳情頁與飼主詳情頁雙向顯示該關聯。
2. Given 寵物已有主要飼主，When 小美將另一位飼主設為主要飼主，Then 原主要飼主自動改為一般關聯，同一寵物任一時點僅有一位主要飼主。

### US-OWN-005 飼主個資遮蔽

> 身為阿豪（單店老闆／OWNER），我想要讓未授權的使用者只能看到遮蔽後的飼主電話與地址，以便符合個資保護要求並降低外洩風險。

- **優先級**：P0｜**對應 FR**：FR-OWN-006

**驗收條件**

1. Given 使用者不具 `owner:pii:read` 權限（如 VIEWER），When 檢視飼主資料，Then 電話與地址以遮蔽格式顯示（如 `0912***678`）。
2. Given 使用者具 `owner:pii:read` 權限，When 檢視同一筆資料，Then 顯示完整個資，且完整個資之讀取行為可被稽核。

### US-OWN-006 軟刪除與還原飼主

> 身為雅婷（租戶管理員／ADMIN），我想要軟刪除不再往來的飼主並在必要時還原，以便清理名單同時保留追溯能力。

- **優先級**：P0｜**對應 FR**：FR-OWN-007

**驗收條件**

1. Given 飼主名下仍有「在店」狀態寵物之關聯，When 雅婷刪除該飼主，Then 系統要求二次確認並明列受影響寵物後才執行軟刪除。
2. Given 飼主已被軟刪除，When 於回收區執行還原，Then 飼主與其寵物關聯一併恢復，且刪除／還原寫入 Audit Log。

### US-OWN-007 管理飼主標籤

> 身為雅婷（租戶管理員／ADMIN），我想要以自訂標籤（VIP、黑名單、待回訪）分類飼主，以便行銷與客服能針對名單分眾操作。

- **優先級**：P1｜**對應 FR**：FR-OWN-008

**驗收條件**

1. Given 雅婷建立自訂標籤，When 為飼主貼上或移除標籤，Then 變更即時反映於飼主列表過濾結果。

### US-OWN-008 合併重複飼主

> 身為雅婷（租戶管理員／ADMIN），我想要將重複建立的飼主合併為一筆並轉移寵物關聯，以便維持客戶資料唯一且完整。

- **優先級**：P1｜**對應 FR**：FR-OWN-009

**驗收條件**

1. Given 兩筆飼主被判定為同一人，When 雅婷執行合併並指定保留主檔，Then 被併飼主之寵物關聯、標籤與互動紀錄轉移至主檔，被併檔案軟刪除。
2. Given 合併已完成，When 查詢 Audit Log，Then 可見合併前後對照（before/after）與操作者。

## 4. 健康管理（HLT）

### US-HLT-001 建立疫苗紀錄

> 身為小美（店員／STAFF），我想要為寵物登記疫苗紀錄（Vaccination）並由系統建議下次到期日，以便不漏打、不重打。

- **優先級**：P0｜**對應 FR**：FR-HLT-001、FR-HLT-002

**驗收條件**

1. Given 小美選擇疫苗種類並輸入施打日，When 儲存紀錄，Then 系統依該疫苗種類規則自動帶入下次到期日，並記錄施打獸醫與疫苗批號。
2. Given 系統已建議下次到期日，When 小美手動覆寫日期，Then 以覆寫值為準並標記為人工調整。
3. Given 施打日晚於今日，When 送出，Then 回傳 422 並提示日期不可為未來。

### US-HLT-002 建立病歷紀錄

> 身為 Dr. Chen（特約獸醫／VET），我想要為診療的寵物建立病歷（MedicalRecord），記錄症狀、診斷、處置與用藥，以便照護有依據並可供後續轉診參考。

- **優先級**：P0｜**對應 FR**：FR-HLT-003

**驗收條件**

1. Given Dr. Chen 對該寵物具診療授權，When 他建立病歷並填寫日期、症狀、診斷、處置、用藥，Then 紀錄建立成功並出現在健康時間軸。
2. Given 病歷建立後需要更正，When Dr. Chen 修改內容，Then 修改成功且 Audit Log 保留 before/after 供醫療紀錄追溯。

### US-HLT-003 記錄一般健康事件

> 身為小美（店員／STAFF），我想要登記驅蟲、健檢、洗牙等一般健康事件，以便疫苗與病歷以外的照護行為也留有紀錄。

- **優先級**：P0｜**對應 FR**：FR-HLT-004

**驗收條件**

1. Given 租戶已設定健康事件類型，When 小美選擇類型並記錄日期與說明，Then 事件建立成功並依時間排入健康時間軸。
2. Given 事件類型清單無所需項目，When 雅婷於設定新增自訂類型，Then 新類型立即可用於記錄。

### US-HLT-004 檢視健康時間軸

> 身為阿豪（單店老闆／OWNER），我想要在單一時間軸整合檢視寵物的疫苗、病歷與健康事件，以便客人或獸醫詢問時能立即回答完整病史。

- **優先級**：P0｜**對應 FR**：FR-HLT-005

**驗收條件**

1. Given 寵物有疫苗、病歷與事件等多類紀錄，When 阿豪開啟健康時間軸，Then 所有紀錄依時間倒序整合呈現並可依類型過濾。
2. Given 某筆健康紀錄已被軟刪除，When 檢視時間軸，Then 該紀錄預設不顯示。

### US-HLT-005 檢視疫苗到期清單

> 身為雅婷（租戶管理員／ADMIN），我想要依 7／30／90 天視窗檢視跨寵物的疫苗到期清單，以便提前安排補打並主動通知飼主。

- **優先級**：P0｜**對應 FR**：FR-HLT-006

**驗收條件**

1. Given 租戶內多隻寵物有未來到期之疫苗，When 雅婷選擇「30 天內到期」視窗，Then 清單彙總顯示到期寵物、疫苗種類、到期日與主要飼主聯絡方式。
2. Given 某疫苗已補打並產生新紀錄，When 重新整理清單，Then 已補打之項目不再出現於到期清單。

### US-HLT-006 上傳健康紀錄附件

> 身為 Dr. Chen（特約獸醫／VET），我想要在健康紀錄附加照片與檢驗報告，以便診斷依據完整留存。

- **優先級**：P0｜**對應 FR**：FR-HLT-007

**驗收條件**

1. Given Dr. Chen 開啟一筆健康紀錄，When 上傳附件（照片或 PDF 檢驗報告），Then 附件存入 R2 並可於該紀錄內預覽或下載。
2. Given 附件格式或大小不符限制，When 上傳，Then 回傳 422 並提示允許之格式與上限。

### US-HLT-007 獸醫授權範圍存取

> 身為阿豪（單店老闆／OWNER），我想要特約獸醫只能存取被授權寵物的健康資料，以便專業協作不犧牲資料邊界。

- **優先級**：P0｜**對應 FR**：FR-HLT-009

**驗收條件**

1. Given Dr. Chen 僅被授權 A、B 兩隻寵物，When 他查詢寵物健康資料，Then 僅能存取 A、B；存取其他寵物回傳 403 或 404。

### US-STO-001 建立與維護門市

> 身為雅婷（租戶管理員／ADMIN），我想要在租戶下建立多個門市（Store）並維護基本資料，以便各分店資料有明確歸屬。

- **優先級**：P1｜**對應 FR**：FR-STO-001、FR-STO-006

**驗收條件**

1. Given 租戶方案允許之門市數尚有餘額，When 雅婷建立新門市，Then 門市建立成功並可於全租戶選單使用。
2. Given 門市數已達方案上限，When 建立新門市，Then 回傳 422 並提示升級方案。

### US-STO-002 資料歸屬門市與過濾

> 身為雅婷（租戶管理員／ADMIN），我想要寵物與使用者可歸屬門市並依門市過濾查詢，以便各店營運狀況能分開檢視。

- **優先級**：P1｜**對應 FR**：FR-STO-002

**驗收條件**

1. Given 寵物已指定所屬門市，When 雅婷於寵物列表選擇門市過濾，Then 僅顯示該門市之寵物。
2. Given 寵物未指定門市（歷史資料），When 過濾「未歸屬」，Then 可列出待補歸屬之資料。

### US-STO-003 門市範圍權限

> 身為雅婷（租戶管理員／ADMIN），我想要將店員的資料範圍限定於所屬門市，以便 A 店店員看不到 B 店的資料。

- **優先級**：P1｜**對應 FR**：FR-STO-003

**驗收條件**

1. Given 小美之權限範圍限定於 A 店，When 她查詢寵物或飼主，Then 僅回傳 A 店資料；直接請求 B 店資源回傳 403/404。
2. Given 雅婷將小美改派至 B 店，When 權限更新生效，Then 小美可見範圍隨之切換，且變更寫入 Audit Log。

### US-STO-004 寵物跨店調撥

> 身為雅婷（租戶管理員／ADMIN），我想要以調撥流程將寵物轉至另一門市並保留歷程，以便帳面歸屬與實際位置一致。

- **優先級**：P1｜**對應 FR**：FR-STO-004

**驗收條件**

1. Given 寵物目前歸屬 A 店，When 雅婷執行調撥至 B 店，Then 寵物歸屬更新為 B 店，調撥歷程記錄時間、經手人與原因。
2. Given 調撥完成，When A 店店員查詢該寵物，Then 依門市權限規則不再可見。

### US-STO-005 跨店儀表板

> 身為雅婷（租戶管理員／ADMIN），我想要在單一儀表板看到各店寵物數、健康到期數與登記案件數，以便巡店前即掌握重點、把管理時間花在刀口上。

- **優先級**：P1｜**對應 FR**：FR-STO-005

**驗收條件**

1. Given 租戶下有多個門市，When 雅婷開啟跨店儀表板，Then 以門市為列呈現寵物數、疫苗到期數（30 天內）與進行中登記案件數，並可點入各店明細。

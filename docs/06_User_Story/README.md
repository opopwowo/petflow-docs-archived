# User Story 使用者故事

> 以使用者視角描述需求與驗收條件。

## 模組介紹

本模組以「身為 <角色>，我想要 <目標>，以便 <效益>」格式記錄需求，遵循 INVEST 原則，每則故事附明確的驗收條件（Acceptance Criteria），作為開發與測試的依據。

## 目的

- 以 INVEST 原則撰寫使用者故事
- 為每則故事定義驗收條件（Given/When/Then）
- 依模組分類管理 Story Backlog
- 連結故事至 Use Case 與測試案例
